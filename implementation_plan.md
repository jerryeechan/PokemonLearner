# Pokemon Vocabulary Database Redesign

## Background

The current [pokemon_vocab.json](file:///Users/jerryee/PokemonLearner/src/data/pokemon_vocab.json) has only **20 entries** across 7 categories (UI, Item, Move, Battle, Status, Town, Dialogue). This is far too few for a year-long learning game. We need a comprehensive database of **500+ entries** sourced from all in-game text content of Pokemon FireRed/LeafGreen (Japanese).

## Proposed Changes

### Data Sourcing Strategy

Since actual web scraping of Bulbapedia/fan wikis is brittle and may violate terms of service, we will use a **hybrid approach**:

1. **Structured Generation Script** — A Node.js script that uses well-known, publicly available Pokemon data (151 Pokemon names, ~165 Gen 1-3 moves, ~50 items, Kanto locations, etc.) to generate a comprehensive vocabulary database
2. **Manual Curated Enrichment** — Each entry gets hand-crafted etymology, explanation, example sentence (with Chinese translation), and difficulty ratings

### New Category System (10 categories)

| Category | Description | Target Count | Difficulty Range |
|----------|-------------|-------------|-----------------|
| **Pokemon** | 151 Gen 1 Pokemon names | ~80 selected | 1-5 |
| **Move** | Battle moves (Gen 1-3 available in FRLG) | ~80 | 1-5 |
| **Item** | Potions, Balls, Key Items, TMs | ~50 | 1-4 |
| **Ability** | Pokemon Abilities (Gen 3) | ~30 | 3-5 |
| **Location** | Towns, routes, landmarks | ~40 | 1-4 |
| **Battle** | Battle system messages & terms | ~40 | 1-4 |
| **UI** | Menu text, interface terms | ~30 | 1-3 |
| **Dialogue** | NPC dialogue phrases, story text | ~50 | 2-5 |
| **Status** | Status conditions, stats, natures | ~30 | 1-4 |
| **Type** | Pokemon types and type-related terms | ~20 | 1-3 |

**Total target: ~450-500 entries**

### Updated Data Schema

```json
{
  "id": "pokemon_025",
  "category": "Pokemon",
  "japanese": "ピカチュウ",
  "kanji": null,
  "hiragana": "ぴかちゅう",
  "katakana": "ピカチュウ",
  "romaji": "pikachuu",
  "zh_tw": "皮卡丘",
  "difficulty": 1,
  "frequency": 5,
  "explanation": "電氣屬性的代表性寶可夢，也是動畫版的主角寶可夢。",
  "etymology": "ピカ來自放電聲「ぴかぴか」，チュウ來自老鼠的叫聲「ちゅう」。",
  "example_sentence": "やせいの ピカチュウが とびだしてきた！",
  "example_sentence_zh": "野生的皮卡丘飛了出來！"
}
```

Key changes from current schema:
- Added `example_sentence_zh` (Chinese translation of example sentence)
- `japanese` field as the primary display form
- `kanji` can be `null` for katakana-only words (like Pokemon names)
- Keep existing fields: `hiragana`, `katakana`, `romaji`, `zh_tw`, `difficulty`, `frequency`, `explanation`, `etymology`
